# Webhook To Solana Example

This example submits a webhook event through ingress and routes it to the Solana adapter by setting `x-intent-kind: solana.transfer.v1`.

## Files

- `webhook-payload.json`: body for `POST /webhooks/:source`
- `run.sh`: Bash end-to-end run
- `run.ps1`: PowerShell end-to-end run

## Run

PowerShell:

```powershell
./examples/webhook_to_solana/run.ps1
```

Bash:

```bash
./examples/webhook_to_solana/run.sh
```

## Optional Env Overrides

- `BASE_URL` (default `http://localhost:8000`)
- `TENANT_ID` (default `tenant_demo`)
- `INGRESS_TOKEN` (default `dev-ingress-token`)
- `STATUS_TOKEN` (default `dev-status-token`)
- `WEBHOOK_SOURCE` (default `demo_partner`)
- `WEBHOOK_INTENT_KIND` (default `solana.transfer.v1`)
- `INGRESS_PRINCIPAL_ID` (default `ingress-service`)
- `WEBHOOK_SUBMITTER_KIND` (default `internal_service`)
- `STATUS_PRINCIPAL_ID` (default `demo-operator`)
- `STATUS_PRINCIPAL_ROLE` (default `admin`)
- `WEBHOOK_PAYLOAD` (default `webhook-payload.json`)
- `WEBHOOK_ID` (default autogenerated)
- `WEBHOOK_SECRET` (optional; if set, script computes `x-webhook-signature`)
- `WEBHOOK_SIGNATURE` (optional explicit `x-webhook-signature`, overrides computed value)

Note: with local compose defaults, principal bindings map `ingress-service` to `internal_service`. If you use `signed_webhook_sender`, update ingress principal bindings accordingly.
